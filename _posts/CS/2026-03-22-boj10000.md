# 쌉소리
접는다고 하고 나서 실제로 거의 한 게 없어요  
요즘은 반쯤 수학과의 정체성을 가지며 살아가는 중입니다

# 문제 이해하기
원이 **x축 위에** 있으며 **교차하지 않**습니다.  
원 개수는 30만이고, 이 원들로 평면이 몇 개의 영역으로 나누어지는지를 구해야 합니다.  

# 문제 풀기
일단 한꺼번에 보지 말고 원을 incremental하게 추가한다고 생각해봅시다.  
원을 하나 추가했을 때 일반적인 경우 평면은 둘로 나누어집니다. 즉, 영역 개수가 하나 늘어납니다.  
그렇지 않은 경우는 어떤 경우일까요?  
문제 지문에 주어진 그림을 살펴봅시다.  
가장 큰 원 빼고 모든 원이 있는 상태에서 가장 큰 원을 추가한다고 가정하면 영역 개수가 둘 늘어날 것입니다.  
이 경우 말고 다른 경우가 있을까요?  
물론 있습니다. 추가되는 순서가 다르면 같은 그림에서 굳이 가장 큰 원이 추가되는 경우가 아니어도 원이 영역을 두 개 늘리는 경우가 생기겠죠.  
우선 이런 경우를 제외시키기 위해, 원들을 반지름의 오름차순으로 정렬해줄 겁니다.  
```py
import sys
input_ = sys.stdin.readline
def minput(): return map(int, input_().split())


N = int(input_())
circles_input = []
for _ in range(N):
    x, r = minput()
    circles_input.append([r, x])
circles_input.sort()
```
이제, 어떤 원이 추가되었을 때 영역을 두 개 늘린다는 것은, 해당 원의 지름이 이미 다른 원들로 완전히 꽉 차 있는 것과 동치입니다.  
이를 확인하기 전에 우선 (중심, 반지름) 꼴로 되어 있는 데이터를 (왼쪽, 오른쪽) 형태로 바꿔 주겠습니다.  
```py
circles_raw = []
coordinates_raw = []
for r, x in circles_input:
    p, q = x - r, x + r
    coordinates_raw.extend([p, q])
    circles_raw.append([p, q])
```
원들로 꽉 차 있는 구간을 하나로 보기 위해 유니온 파인드를 사용할 겁니다.  
작은 원이 하나 추가되었다고 하면 그 시작부터 끝까지를 union해 줍니다.  
큰 원이 들어왔을 때 그 원이 영역을 두 개 만드는지 보고 싶을 때는 find(left) == find(right)과 같은 식을 써 주고 싶습니다.  
그러므로 바로 왼쪽 좌표와 바로 오른쪽 좌표 또한 원이 먹고 있는지 확인한 뒤, 먹고 있는 경우 union해 줍니다.  
여기까지 들었을 때에는 O(N^2)가 걸리겠다는 생각이 들지만, 원끼리 교차하지 않는다는 조건이 빛을 발하게 됩니다.  
원끼리 교차하지 않는다는 조건 덕에, 큰 원이 추가되었을 때 작은 원의 안쪽까지 들여다볼 이유가 없습니다.  
그러므로 union할 때도 시작점과 끝점만 union해주면 되고, 바로 왼쪽과 바로 오른쪽도 어차피 원이 먹고 있다면 원의 끝부분이 먹고 있을 것입니다.  
점을 원이 먹고 있는지 안 먹고 있는지는 그냥 배열로 확인해줄 건데, 여기서도 원 하나당 최대 2개의 값을 업데이트하고 2개의 값을 확인하므로 시간상 문제가 없게 됩니다.
```py
rank = coordinate_compression(coordinates_raw)
circles = []
M = 0
for p, q in circles_raw:
    x, y = rank[p] * 2 + 1, rank[q] * 2 + 1
    circles.append([x, y])
    M = max(M, x, y)

set_bit = [0] * (M + 1)
uf = UnionFind(M + 1)
ans = 1
for l, r in circles:
    ans += 1 + (uf.find(l) == uf.find(r - 1))
    if set_bit[l - 1]:
        uf.union(l - 1, l)
    if set_bit[r]:
        uf.union(r - 1, r)
    set_bit[l] = set_bit[r - 1] = 1
    uf.union(l, r - 1)

print(ans)
```
첫 파트는 단순히 좌표압축을 해 주지만, p, q의 rank값을 그대로 넣는 것이 아니고 2배한 뒤 1을 더해줍니다.  
이는 가장 작은 원의 경우 l == r - 1이 되기 때문에 가장 작은 원이 영역을 2개 만든다고 오판하기 떄문입니다.  
두 번째가 본문에서 설명한 유니온 파인드를 그대로 구현하는 과정입니다.  
역시나 구현 디테일로, 양끝점이 l, r인 원은 [l, r) 구간을 먹는다고 생각하시면 됩니다.

# 코드
```py
import sys
input_ = sys.stdin.readline
def minput(): return map(int, input_().split())


class UnionFind:
    def __init__(self, x):
        self.parent = [i for i in range(x)]
        self.size = [1] * x

    def find(self, x):
        if self.parent[x] != x:
            self.parent[x] = self.find(self.parent[x])
        return self.parent[x]

    def union(self, x, y):
        root_x = self.find(x)
        root_y = self.find(y)
        if root_x == root_y:
            return
        elif self.size[root_x] >= self.size[root_y]:
            self.size[root_x] += self.size[root_y]
            self.parent[root_y] = root_x
        else:
            self.size[root_y] += self.size[root_x]
            self.parent[root_x] = root_y


def coordinate_compression(lst):
    distinct = sorted(set(lst))
    rank = {distinct[i]: i for i in range(len(distinct))}
    return rank


N = int(input_())
circles_input = []
for _ in range(N):
    x, r = minput()
    circles_input.append([r, x])
circles_input.sort()

circles_raw = []
coordinates_raw = []
for r, x in circles_input:
    p, q = x - r, x + r
    coordinates_raw.extend([p, q])
    circles_raw.append([p, q])

rank = coordinate_compression(coordinates_raw)
circles = []
M = 0
for p, q in circles_raw:
    x, y = rank[p] * 2 + 1, rank[q] * 2 + 1
    circles.append([x, y])
    M = max(M, x, y)

set_bit = [0] * (M + 1)
uf = UnionFind(M + 1)
ans = 1
for l, r in circles:
    ans += 1 + (uf.find(l) == uf.find(r - 1))
    if set_bit[l - 1]:
        uf.union(l - 1, l)
    if set_bit[r]:
        uf.union(r - 1, r)
    set_bit[l] = set_bit[r - 1] = 1
    uf.union(l, r - 1)

print(ans)
```
